# Configuration

## MCP Client Configuration

The mem/ctl MCP server is configured through environment variables passed via your MCP client configuration.

### Claude Code

Edit `~/.claude/claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "memctl": {
      "command": "npx",
      "args": ["@memctl/cli"],
      "env": {
        "MEMCTL_TOKEN": "mctl_...",
        "MEMCTL_ORG": "my-team",
        "MEMCTL_PROJECT": "my-app"
      }
    }
  }
}
```

### Cursor

Create `.cursor/mcp.json` in your project root:

```json
{
  "mcpServers": {
    "memctl": {
      "command": "npx",
      "args": ["@memctl/cli"],
      "env": {
        "MEMCTL_TOKEN": "mctl_...",
        "MEMCTL_ORG": "my-team",
        "MEMCTL_PROJECT": "my-app"
      }
    }
  }
}
```

### Per-Project Configuration

You can use different projects for different repositories by setting `MEMCTL_PROJECT` to different values in each project's MCP config.

## Self-Hosting

When self-hosting, set `MEMCTL_API_URL` to point to your own instance:

```json
{
  "env": {
    "MEMCTL_API_URL": "https://your-instance.com/api/v1"
  }
}
```
