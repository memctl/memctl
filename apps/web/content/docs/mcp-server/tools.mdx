# MCP Tools

mem/ctl exposes the following tools via the Model Context Protocol.

## memory_store

Store a key-value memory for the current project. If a memory with the same key exists, it will be updated.

**Parameters:**

| Name | Type | Required | Description |
|---|---|---|---|
| `key` | string | Yes | Unique key for the memory (max 256 chars) |
| `content` | string | Yes | Content to store (max 65,536 chars) |
| `metadata` | object | No | Optional metadata object |

## memory_get

Retrieve a single memory by its key.

**Parameters:**

| Name | Type | Required | Description |
|---|---|---|---|
| `key` | string | Yes | Key of the memory to retrieve |

## memory_search

Search memories by query string. Searches both keys and content.

**Parameters:**

| Name | Type | Required | Description |
|---|---|---|---|
| `query` | string | Yes | Search query |
| `limit` | number | No | Max results (default: 20, max: 100) |

## memory_list

List all memories for the current project.

**Parameters:**

| Name | Type | Required | Description |
|---|---|---|---|
| `limit` | number | No | Max results (default: 100, max: 100) |
| `offset` | number | No | Pagination offset (default: 0) |

## memory_delete

Delete a memory by its key.

**Parameters:**

| Name | Type | Required | Description |
|---|---|---|---|
| `key` | string | Yes | Key of the memory to delete |

## memory_update

Update an existing memory's content or metadata.

**Parameters:**

| Name | Type | Required | Description |
|---|---|---|---|
| `key` | string | Yes | Key of the memory to update |
| `content` | string | No | New content |
| `metadata` | object | No | New metadata |
